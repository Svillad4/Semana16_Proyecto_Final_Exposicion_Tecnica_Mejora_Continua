# 08 - Rubrica de evaluacion sugerida

| Criterio | Porcentaje | Nivel esperado |
|---|---:|---|
| Proyecto ejecutable y funcional | 20% | El proyecto corre con `npm run start`, abre en navegador y conecta con JSON Server. |
| Validaciones y pruebas finales | 15% | El formulario valida datos y se evidencian pruebas funcionales. |
| Evidencias organizadas | 15% | Las capturas estan en carpetas correctas y demuestran funcionamiento. |
| Documentacion tecnica | 15% | README y documentos explican instalacion, ejecucion, estructura y funciones. |
| Argumentacion de decisiones | 15% | El estudiante justifica herramientas, estructura y soluciones usadas. |
| Guion de exposicion | 10% | La exposicion tiene orden, claridad y demostracion funcional. |
| Plan de mejora continua | 10% | Propone mejoras futuras justificadas y posibles de implementar. |

## Nivel superior

El estudiante entrega un proyecto funcional, probado, documentado y preparado para exposicion. Justifica con claridad sus decisiones tecnicas, presenta evidencias organizadas y propone mejoras coherentes.

## Nivel alto

El estudiante entrega un proyecto funcional y documentado, con evidencias suficientes y exposicion clara, aunque puede profundizar mas en la argumentacion.

## Nivel basico

El estudiante logra ejecutar el proyecto, pero presenta documentacion incompleta, pocas evidencias o exposicion poco estructurada.

## Nivel bajo

El estudiante no demuestra funcionamiento completo, no organiza evidencias y no logra explicar decisiones tecnicas.
